# 小麦病虫害智能问答与学习平台 - 数据库设计文档

---

## 1. 数据库概述

### 1.1 数据库选型

系统采用 **PostgreSQL 16.x** 作为主数据库，配合 **pgvector** 扩展支持向量相似度检索。选择理由如下：

| 特性     | PostgreSQL       | 选型理由                              |
| -------- | ---------------- | ------------------------------------- |
| 向量支持 | pgvector扩展     | 支持向量存储和相似度检索，满足RAG需求 |
| 事务支持 | 完整ACID         | 保证数据一致性                        |
| 扩展性   | 支持分区表       | 支持大规模数据存储                    |
| 全文检索 | tsvector/tsquery | 支持文本检索                          |
| JSON支持 | JSONB类型        | 灵活存储半结构化数据                  |

### 1.2 数据库架构

```text
┌─────────────────────────────────────────────┐
│              PostgreSQL 16.x                │
│  ┌───────────────────────────────────────┐  │
│  │        业务数据表（关系型）           │  │
│  │  users, admins, conversations...     │  │
│  └───────────────────────────────────────┘  │
│                    │                       │
│  ┌─────────────────┴─────────────────┐     │
│  │          pgvector扩展             │     │
│  │   knowledge_chunks.embedding      │     │
│  └───────────────────────────────────┘     │
└─────────────────────────────────────────────┘
```

## 2. 数据表设计

### 2.1 用户表 (users)

| 字段名        | 类型         | 约束                      | 说明               |
| ------------- | ------------ | ------------------------- | ------------------ |
| id            | BIGSERIAL    | PRIMARY KEY               | 用户唯一标识       |
| email         | VARCHAR(100) | UNIQUE NOT NULL           | 用户邮箱           |
| password_hash | VARCHAR(255) | NOT NULL                  | 密码哈希值(bcrypt) |
| nickname      | VARCHAR(50)  | NULL                      | 用户昵称           |
| avatar        | VARCHAR(255) | NULL                      | 头像URL            |
| created_at    | TIMESTAMP    | DEFAULT CURRENT_TIMESTAMP | 创建时间           |
| updated_at    | TIMESTAMP    | DEFAULT CURRENT_TIMESTAMP | 更新时间           |
| is_deleted    | BOOLEAN      | DEFAULT FALSE             | 是否删除           |

**索引设计**:

- `idx_users_email`: email字段唯一索引
- `idx_users_is_deleted`: is_deleted字段普通索引

- `idx_admins_username`: username字段唯一索引

### 2.6 题库表 (questions)

| 字段名      | 类型        | 约束                      | 说明                    |

| id          | BIGSERIAL   | PRIMARY KEY               | 题目唯一标识            |

| options     | JSONB       | NOT NULL                  | 选项（JSON数组）        |

**索引设计**:

- `idx_questions_category_id`: category_id字段普通索引

**外键约束**:

- `category_id` REFERENCES `pest_categories(id)`

### 2.9 知识分块表 (knowledge_chunks)

| 字段名      | 类型         | 约束                      | 说明                     |

| updated_at  | TIMESTAMP    | DEFAULT CURRENT_TIMESTAMP | 更新时间                 |

**索引设计**:

- `idx_knowledge_chunks_is_deleted`: is_deleted字段普通索引

- `document_id` REFERENCES `knowledge_documents(id)` ON DELETE CASCADE

- 当前实现：存储JSON数组字符串格式
- 后续优化：安装pgvector后可升级为`VECTOR(1536)`类型

## 4. 数据库优化建议

### 4.1 索引优化

- 为频繁查询的字段创建索引
- 定期分析查询性能，调整索引策略
- 避免过多索引影响写入性能

### 4.2 向量检索优化

- 安装pgvector扩展后，将embedding字段类型从TEXT改为VECTOR(1536)
- 使用HNSW索引优化向量相似度检索性能
- 设置合适的相似度阈值和召回数量

### 4.3 数据清理策略

- 定期清理已删除的会话和消息
- 归档历史数据到冷存储
- 设置合理的数据保留期限

### 4.4 备份策略

- 每日全量备份数据库
- 实时增量备份（WAL日志）
- 备份数据异地存储
