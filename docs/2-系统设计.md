### 2.2 核心技术说明

#### 2.2.1 RAG架构

系统采用检索增强生成（RAG）技术，核心流程如下：

1. **文档预处理**：上传文档 → 文本切片 → Embedding生成 → 向量存储
2. **查询处理**：用户提问 → 查询向量化 → 相似度检索 → 知识召回
3. **回答生成**：召回知识 + 用户问题 → LLM生成 → 返回结果

#### 2.2.2 向量数据库

使用PostgreSQL + pgvector扩展存储向量数据：

- 支持高效的向量相似度检索（如余弦相似度）
- 支持大规模数据存储
- 与关系数据共存，便于事务管理

## 4. 关键设计

### 4.1 认证设计

#### 4.1.1 JWT认证流程

```text
用户登录 → 验证凭证 → 生成Access Token + Refresh Token
                ↓
           返回Token给客户端
                ↓
       后续请求携带Access Token
                ↓
       服务端验证Token有效性
                ↓
           处理业务请求
```

#### 4.1.2 Token策略

- **Access Token**：有效期15分钟，用于API请求认证
- **Refresh Token**：有效期7天，用于刷新Access Token
- Token存储：客户端存储在HttpOnly Cookie中

#### 4.1.3 权限控制

基于角色的访问控制（RBAC）：

| 角色     | 权限                                     |
| -------- | ---------------------------------------- |
| 游客     | AI问答（限制次数）、浏览知识             |
| 普通用户 | AI问答、历史会话、知识答题、错题本       |
| 管理员   | 知识库管理、题库管理、用户管理、数据统计 |

### 4.2 RAG流程设计

#### 4.2.1 文档处理流程

```text
文档上传 → 格式解析 → 文本提取 → 文本切片 → Embedding生成 → 向量存储
     │                │           │          │              │
     ▼                ▼           ▼          ▼              ▼
 支持多格式       PDF/DOCX    纯文本提取   按语义切分   BGE-M3模型   PostgreSQL+pgvector
  PDF/DOCX/TXT/MD
```

#### 4.2.2 问答流程

```text
用户提问 → 查询向量化 → 相似度检索 → Top-K召回 → LLM生成 → 返回结果
     │              │            │          │          │          │
     ▼              ▼            ▼          ▼          ▼          ▼
 自然语言      BGE-M3模型   pgvector     知识库文档   DeepSeek   结构化响应
                           余弦相似度
```

### 4.3 会话管理设计

#### 4.3.1 会话结构

```text
Conversation（会话）
├── id: string
├── userId: string
├── title: string（首条问题摘要）
├── messages: Message[]
├── createdAt: Date
└── updatedAt: Date

Message（消息）
├── id: string
├── conversationId: string
├── role: 'user' | 'assistant'
├── content: string
├── promptTokens: number
├── completionTokens: number
└── createdAt: Date
```

#### 4.3.2 上下文管理

- 每个会话维护独立的对话历史
- 支持多轮对话，自动拼接历史上下文
- 会话过期策略：30天未访问自动清理

### 4.4 答题流程设计

#### 4.4.1 答题流程

```text
获取题目 → 用户作答 → 提交答案 → 自动判题 → 记录结果 → 返回解析
     │           │          │          │          │          │
     ▼           ▼          ▼          ▼          ▼          ▼
 随机/专项    选择题/      提交请求    比对答案    答题记录    AI生成解析
             多选题                    统计正确性
```

#### 4.4.2 错题管理

- 自动记录答错的题目
- 支持按分类筛选错题
- 提供错题复习功能

### 4.5 安全性设计

#### 4.5.1 数据安全

- 用户密码使用bcrypt加密存储
- 所有API请求使用HTTPS传输
- 敏感数据传输加密

#### 4.5.2 攻击防护

- 防止SQL注入（Prisma参数化查询）
- 防止XSS攻击（输入过滤、输出转义）
- 防止CSRF攻击（JWT + 同源策略）
- API限流（防止暴力请求）

#### 4.5.3 日志审计

- 记录关键操作日志
- 支持日志检索与分析
- 异常行为告警
