## 1. 部署概述

### 1.1 部署架构

系统采用容器化部署方案，使用 Docker Compose 管理多个服务。整体架构如下：

```text
Internet
    │
    ▼
┌──────────────────────────────────────┐
│              Nginx                   │
│  (反向代理、负载均衡、静态资源)        │
└──────────────┬───────────────────────┘
               │
    ┌──────────┼──────────┐
    ▼          ▼          ▼
┌─────────┐ ┌─────────┐ ┌─────────┐
│Frontend │ │  Admin  │ │  API    │
│(Next.js)│ │(Next.js)│ │(Express)│
└────┬────┘ └────┬────┘ └────┬────┘
     │           │           │
     └───────────┴─────┬─────┘
                       ▼
              ┌─────────────┐
              │  PostgreSQL │
              │   +pgvector │
              └──────┬──────┘
                     │
              ┌──────┴──────┐
              │   Redis     │
              │  (缓存)     │
              └─────────────┘
```

### 1.2 服务列表

| 服务       | 容器名称 | 端口   | 说明        |
| ---------- | -------- | ------ | ----------- |
| Nginx      | nginx    | 80/443 | 反向代理    |
| Frontend   | frontend | 3000   | 用户端应用  |
| Admin      | admin    | 3001   | 管理后台    |
| API        | api      | 4000   | 后端API服务 |
| PostgreSQL | postgres | 5432   | 主数据库    |
| Redis      | redis    | 6379   | 缓存服务    |

## 3. 开发环境部署

### 3.1 环境准备

```bash
# 克隆项目
git clone <repository-url>
cd WPD-QA

# 安装所有项目依赖
pnpm install:all

# 或分别安装
cd frontend && pnpm install
cd admin && pnpm install
cd backend && pnpm install

# 创建环境变量文件
cp backend/.env.example backend/.env
```

### 3.2 环境变量配置

编辑 `backend/.env` 文件：

```env
# 数据库配置
DATABASE_URL=postgresql://postgres:password@localhost:5432/wpd_qa

# Redis配置
REDIS_URL=redis://localhost:6379

# JWT密钥（使用openssl rand -hex 32生成）
JWT_SECRET=your_jwt_secret_key_here

# DeepSeek API配置
DEEPSEEK_API_KEY=your_deepseek_api_key
DEEPSEEK_API_URL=https://api.deepseek.com/v1

# 端口配置
PORT=4000
```

### 3.3 启动开发服务器

```bash
# 启动数据库和缓存（使用Docker）
docker-compose up -d postgres redis

# 初始化数据库
pnpm db:generate
pnpm db:migrate

# 启动后端API
pnpm dev:backend

# 启动前端（新终端）
pnpm dev:frontend

# 启动管理后台（新终端）
pnpm dev:admin
```

### 3.4 访问地址

| 服务     | 地址                           |
| -------- | ------------------------------ |
| 用户端   | http://localhost:3000          |
| 管理后台 | http://localhost:3001          |
| API文档  | http://localhost:4000/api/docs |

## 4. Docker Compose 部署

### 4.1 构建镜像

```bash
# 构建所有服务
docker-compose build

# 或单独构建
docker-compose build frontend
docker-compose build admin
docker-compose build api
```

### 4.2 启动服务

```bash
# 启动所有服务（后台运行）
docker-compose up -d

# 查看运行状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 4.3 停止服务

```bash
# 停止所有服务
docker-compose down

# 停止并删除数据卷（谨慎使用）
docker-compose down -v
```

### 4.4 Docker Compose 配置说明

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:16
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      POSTGRES_DB: wpd_qa
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    ports:
      - '5432:5432'
    healthcheck:
      test: ['CMD-SHELL', 'pg_isready -U postgres']
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data
    ports:
      - '6379:6379'
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 5

  api:
    build: ./backend
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
    environment:
      DATABASE_URL: postgresql://postgres:${DB_PASSWORD}@postgres:5432/wpd_qa
      REDIS_URL: redis://redis:6379
      JWT_SECRET: ${JWT_SECRET}
      DEEPSEEK_API_KEY: ${DEEPSEEK_API_KEY}
    ports:
      - '4000:4000'

  frontend:
    build: ./frontend
    depends_on:
      - api
    environment:
      NEXT_PUBLIC_API_URL: http://localhost:4000/api
    ports:
      - '3000:3000'

  admin:
    build: ./admin
    depends_on:
      - api
    environment:
      NEXT_PUBLIC_API_URL: http://localhost:4000/api
    ports:
      - '3001:3001'

volumes:
  postgres_data:
  redis_data:
```

## 5. 生产环境部署

### 5.1 环境准备

#### 5.1.1 服务器配置

- **操作系统**: Ubuntu 22.04 LTS
- **用户**: 创建专用部署用户 `wpd`
- **防火墙**: 开放端口 80, 443, 22

```bash
# 创建用户
sudo useradd -m wpd
sudo usermod -aG sudo wpd
su - wpd

# 安装依赖
sudo apt update && sudo apt install -y docker.io docker-compose-plugin
```

#### 5.1.2 SSL证书配置

使用 Let's Encrypt 获取 SSL 证书：

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 获取证书
sudo certbot certonly --nginx -d example.com -d www.example.com
```

### 5.2 Nginx 配置

创建 `/etc/nginx/sites-available/wpd-qa.conf`：

```nginx
server {
    listen 80;
    server_name example.com www.example.com;

    # HTTP 重定向到 HTTPS
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 用户端
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # 管理后台
    location /admin/ {
        proxy_pass http://localhost:3001/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # API
    location /api/ {
        proxy_pass http://localhost:4000/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/wpd-qa.conf /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 5.3 环境变量配置

创建生产环境 `.env` 文件（位于 `backend/.env`）：

```env
# 数据库配置
DATABASE_URL=postgresql://postgres:secure_password@postgres:5432/wpd_qa
DB_PASSWORD=secure_password

# Redis配置
REDIS_URL=redis://redis:6379

# JWT密钥（生产环境使用强密钥）
JWT_SECRET=your_strong_production_jwt_secret_here

# DeepSeek API配置
DEEPSEEK_API_KEY=your_production_deepseek_api_key
DEEPSEEK_API_URL=https://api.deepseek.com/v1

# 环境标识
NODE_ENV=production

# CORS配置
ALLOWED_ORIGINS=https://example.com

# 日志级别
LOG_LEVEL=info
```

### 5.4 启动生产服务

```bash
# 使用生产环境变量启动
docker-compose --env-file backend/.env up -d

# 查看运行状态
docker-compose ps
```

## 6. 数据库配置

### 6.1 Prisma 配置

Prisma Schema 位于：`backend/prisma/schema.prisma`

### 6.2 PostgreSQL 初始化

```bash
# 进入数据库容器
docker exec -it wpd-qa-postgres-1 psql -U postgres -d wpd_qa

# 创建扩展
CREATE EXTENSION IF NOT EXISTS vector;

# 退出
\q
```

### 6.3 数据库备份

```bash
# 创建备份
docker exec wpd-qa-postgres-1 pg_dump -U postgres wpd_qa > backup_$(date +%Y%m%d).sql

# 恢复备份
docker exec -i wpd-qa-postgres-1 psql -U postgres wpd_qa < backup_20240101.sql
```

### 6.4 定期备份脚本

创建 `/home/wpd/backup.sh`：

```bash
#!/bin/bash
BACKUP_DIR="/home/wpd/backups"
DATE=$(date +%Y%m%d_%H%M%S)
CONTAINER_NAME="wpd-qa-postgres-1"

mkdir -p $BACKUP_DIR

docker exec $CONTAINER_NAME pg_dump -U postgres wpd_qa > "$BACKUP_DIR/backup_$DATE.sql"

# 保留最近7天的备份
find $BACKUP_DIR -name "backup_*.sql" -mtime +7 -delete
```

添加定时任务：

```bash
# 每天凌晨2点备份
crontab -e
# 添加：0 2 * * * /home/wpd/backup.sh
```

## 7. 安全配置

### 7.1 防火墙配置

```bash
# 允许必要端口
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

### 7.2 安全最佳实践

| 措施           | 说明                                                   |
| -------------- | ------------------------------------------------------ |
| 禁用 root 登录 | 修改 `/etc/ssh/sshd_config`，设置 `PermitRootLogin no` |
| 使用密钥登录   | 禁用密码登录，使用 SSH 密钥                            |
| 定期更新系统   | `sudo apt update && sudo apt upgrade -y`               |
| 限制容器权限   | 使用非 root 用户运行容器                               |
| 加密敏感数据   | 数据库密码、API密钥使用环境变量                        |

## 9. 故障排查

### 9.1 常见问题

| 问题              | 原因                   | 解决方案                       |
| ----------------- | ---------------------- | ------------------------------ |
| API无法连接数据库 | 数据库未启动或配置错误 | 检查数据库容器状态和连接字符串 |
| 前端无法访问API   | CORS配置错误           | 检查ALLOWED_ORIGINS环境变量    |
| 页面加载缓慢      | 资源未优化             | 确保启用生产模式构建           |
| 数据库连接池耗尽  | 连接数配置不足         | 调整数据库连接池大小           |
| JWT验证失败       | 密钥不一致             | 检查JWT_SECRET环境变量         |

### 9.2 日志排查

```bash
# 查看错误日志
docker-compose logs api 2>&1 | grep -i error

# 查看警告
docker-compose logs api 2>&1 | grep -i warn
```
